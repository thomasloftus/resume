üíæ Key Data Concepts

Normalization: Organizing data to reduce redundancy and improve data integrity.
  • 1NF: Atomic values, no repeating groups.
  • 2NF: 1NF + No partial dependencies (non-key attributes depend on the *entire* PK).
  • 3NF: 2NF + No transitive dependencies (non-key attributes don't depend on other non-key attributes).
Referential Integrity: Ensuring relationships between tables are valid (FKs point to existing PKs). Prevents orphaned records.
-- Example SQL for Referential Integrity
ALTER TABLE Orders
ADD FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID);
Data Types: Choosing appropriate types (int, string, date, boolean, etc.) for efficiency and accuracy (See Field Types section).

üßæ Field Types (Common)

int: Stores whole numbers (integers).
  • Use Cases: Unique ID fields (Primary Keys), quantities, counts (e.g., number of orders).
  • Considerations: Choose appropriate size (e.g., TINYINT, INT, BIGINT) based on expected range of values.
string/varchar: Stores variable-length text strings. `VARCHAR(n)` specifies maximum length `n`.
  • Use Cases: Names, emails, addresses, phone numbers, product descriptions, notes.
  • Considerations: `VARCHAR` is generally preferred over fixed-length `CHAR` for efficiency when string lengths vary significantly.
float/decimal: Stores numbers with decimal points.
  • Use Cases: Prices, tax rates, discount percentages, calculated metrics.
  • Considerations: Use `DECIMAL` for currency or financial data requiring exact precision; `FLOAT` can have rounding issues. Specify precision and scale for `DECIMAL`.
date/datetime: Stores date and/or time values.
  • Use Cases: Order dates, creation timestamps (CreatedAt), due dates, last contact date.
  • Considerations: `DATE` stores only the date, `TIME` only time, `DATETIME` or `TIMESTAMP` store both. Choose based on required granularity.
boolean: Stores true or false values. Often represented as 0 or 1 in databases.
  • Use Cases: Flags or switches (e.g., IsActive, IsArchived, HasOptedIn).
  • Considerations: Simple and efficient for binary states.
-- Example SQL Table Definition Snippet with Data Types
CREATE TABLE Customers (
CustomerID INT PRIMARY KEY,
Name VARCHAR(255),
Email VARCHAR(255) UNIQUE,
RegistrationDate DATE,
LastLogin DATETIME,
TotalSpent DECIMAL(10, 2),
IsActive BOOLEAN
);

  2. System Design: Configure CRM modules, customize fields, design workflows.

üîÑ The CRM Process/Lifecycle

  1. Generate Awareness: Introduce potential customers to your brand.
  2. Acquire Leads: Gather information from interested prospects.
  3. Convert Leads: Nurture leads and turn them into paying customers.
  4. Provide Service & Support: Assist customers and resolve issues post-purchase.
  5. Drive Loyalty & Retention: Build strong relationships for repeat business.
  6. Upsell/Cross-sell: Identify opportunities to sell more to existing customers.
